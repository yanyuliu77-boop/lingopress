# LingoPress

每日外刊精读的静态 Web 应用。首篇内容基于用户提供的义乌外刊精读 PDF，包含三种阅读模式、浏览器英文朗读、生词解析与收藏、按文章自动保存笔记、阅读进度、理解测验、连续学习记录、往期内容和 PWA 离线缓存。

## 每日 07:00 自动更新

`.github/workflows/daily-edition.yml` 会在北京时间 06:30 预生成当天精读，并在 06:50、07:10 自动补跑。内容文件预先写入当天 `07:00 +08:00` 的发布时间，网站只会在该时间到达后展示，因此不需要人工上传新稿。

发布器 `scripts/publish_daily.py` 轮换读取 BBC、The Guardian、The New York Times 等公开 RSS，从未使用过的最新报道中选题，再通过 GitHub Models 生成原创教学改写。每期包括：

- 520-680 词 C1 英文精读与逐段中文翻译
- 6 个写作、语法或衔接知识点
- 12-15 个重点词汇、音标、释义和例句
- 当日短语、长难句拆解和 3 道理解题
- 原报道链接以及 Wikimedia Commons 图片署名

网站不会公开复制付费报刊全文。报道仅作为事实材料，发布内容为原创教学改写，并保留来源链接。内容库保留最近 60 期；读者可从“往期”打开任意一期。

工作流也支持在 GitHub Actions 页面手动执行 `workflow_dispatch`。若新闻源、模型或图片服务短暂失败，补跑任务会再次尝试，旧一期在新稿成功前保持可用。

## 访问密钥与店铺自动发货

首批 20 个访问密钥位于未参与部署的 `private/自动发货密钥.csv`。将“网站链接 + 一枚密钥”作为卡密商品导入淘宝或小红书店铺使用的自动发货工具即可。浏览器只接收密钥的 SHA-256 哈希，首次验证后在本机保存授权。

这是一套适合首发的轻量卡密方案。业务规模扩大后，应把校验迁移到服务端数据库，以支持一单一码核销、设备数限制、退款撤销与后台补发。

## 本地运行

```sh
python3 -m http.server 4173
```

打开 `http://localhost:4173`。文章内容位于 `content/articles.js`，页面会根据北京时间和每期的 `publishAt` 自动选择今日内容。
